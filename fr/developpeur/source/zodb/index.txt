===================================================
ZODB - une base de données objet native pour python
===================================================

  N'éclatez pas vos objets dans des tables : stockez les dans une base objet.

Résumé
======

Les programmes Python sont généralement écrits en suivant le paradigme orienté
objet. Vous pouvez utiliser des objets qui se références les uns les autres sans
contraintes de type ou d'architecture : aucun objet n'est lié à un schéma
spécifique et peut porter n'importe quelle information.

Stocker ces objets dans une base de données relationnelle exige que vous
sacrifiez votre liberté d'architecture et la puissance des références.  Les
contraintes des bases relationnelles réduisent votre capacité à écrire du code
orienté objet.

La ZODB est une base de données objet native, qui stocke vos objets et vous
permet de travailler selon n'importe quel des paradigmes possibles en Python. En
conséquence votre code est plus simple, plus robuste et compréhensible.

De plus il n'y a pas de gap entre la base de donnée et le programme : pas de
code servant de colle à écrire, ni de mappings à configurer. Regardez le
tutoriel pour voir au combien c'est facile à utiliser.

Voici quelles que unes des fonctionnalités que la ZODB vous apporte :

* Persistance transparente pour les objets Python.
* Support des transactions complètement ACID (avec les point de sauvegardes).
* Historique et capacité de défaire refaire (undo).
* Support efficace pour les objets binaires volumineux (BLOBs).
* Dépôts pluggable.
* Architecture adaptable.

Téléchargements
===============

ZODB est distribué sous forme d'un egg (paquet) Python accessible sur le dépôt
de la communauté Python `Python Package Index
<http://pypi.python.org/pypi/ZODB3>`_.

Il est à la base de Zope et donc de Plone.

Vous pouvez l'installer seul à l'aide de la commande easy_install 'setuptools'
: :: 

    $ easy_install ZODB3

L'installation des anciennes versions sont possibles via le gestionnaire de
version Subversion.

Communauté et contribution
==========================

Les discussions ont lieux sur la mailing liste `ZODB developers' mailing list
<http://mail.zope.org/mailman/listinfo/zodb-dev>`_.

:doc:`Bug reporting<bugs>`, :doc:`feature requests<features>`, et le planning
des version sont réalisés sur `Launchpad <http://launchpad.net/zodb>`_.

Si vous voulez contribuer la communauté sera heureuse de recevoir de l'aide sur
la documentation, l'aide des autres développeurs et des utilisateurs sur la
mailing liste, soumission de rapport de bogues, proposition d'amélioration et
écriture de code.

ZOD est un projet géré par la fondation Zope où vous pouvez obtenir les droits
pour contribuer directement sur le code - pour cela récupérer la documentation
de la communauté `Zope Developer Information <http://docs.zope.org/developer>`_.
